    操作マニュアル¶

    このページは 既にセットアップ済のユーザが日常使いで参照する 操作手順です。初回導入は セットアップガイド を参照してください。

    ツールはシート単位で機能が分かれており、メインシートの各ボタンが対応するシートへ遷移します。

    シート一覧と役割¶

    シート名 役割
    メイン 各機能へのエントリポイント。全ボタンが揃う
    ナレッジ登録 新規ナレッジを 1 件入力して登録
    ナレッジ修正 既存ナレッジ番号を指定してフィールド値を編集
    ナレッジ一覧 登録済の全ナレッジを一覧表示
    検索 キーワード + フォーマット指定で検索を実行、結果一覧を表示
    フォーマット設計 新規フォーマット（フィールド構成）を定義
    フォーマット一覧 登録済フォーマットの一覧
    フォーマットプレビュー 選択中フォーマットの入力フォーム見本を表示
    既存データへのフィールド反映 フォーマット定義変更を既存ナレッジに伝播
    格納先設定 ナレッジテキスト・画像の保存先フォルダを指定
    設定 テストモード等のシステム設定
    ログ 実行ログを確認

    メインシート以外は通常「非表示」になっており、メインの該当ボタンを押すと表示・遷移します。元に戻るには各シートの [戻る] または [メインへ] ボタンを押します。

    ナレッジ登録¶

    何ができるか¶

    新規ナレッジを 1 件入力して登録します。フォーマット (FormatID) を指定し、そのフォーマットに紐づくフィールドへ値を入れて保存すると、ナレッジ番号が自動採番されてナレッジ一覧に追加されます。

    標準操作手順¶

    1. メインシートの [ナレッジ登録] ボタンを押す → ナレッジ登録シートへ遷移
    2. FormatID セルに使用するフォーマット ID を入力（例: DEMO-MEMO）
    3. [フォーマット読込] ボタンを押す → そのフォーマットのフィールド一覧が画面に展開される
    4. 各フィールドに値を入力
      • text 系: 直接入力
      • image 系: 画像ファイルを kb_images/ 配下に置いて、ファイル名を ImagePath フィールドに指定
      • date 系: YYYY-MM-DD 形式で入力
    5. [登録] ボタンを押す → 自動採番された KnwNo で保存され、ナレッジ一覧に 1 行追加
    6. 完了メッセージが出たら [戻る] でメインへ

    押すボタン¶

    ボタン 動作
    [フォーマット読込] 指定 FormatID のフィールド構成を画面に展開
    [登録] 入力値を検証 → 保存 → ナレッジ一覧へ追加
    [クリア] 入力欄を初期化
    [戻る] メインシートへ戻る

    ナレッジ修正¶

    何ができるか¶

    既存ナレッジ番号 (KnwNo) を指定して、そのナレッジのフィールド値を編集します。

    標準操作手順¶

    1. メインシートの [ナレッジ修正] ボタンを押す → ナレッジ修正シートへ遷移
    2. KnwNo セルに修正対象のナレッジ番号を入力（例: KN-2026-0420）
    3. [読込] ボタンを押す → 該当ナレッジの現在値が画面に展開
    4. 修正したいフィールドの値を上書き
    5. [更新] ボタンを押す → 保存 → ナレッジ一覧の該当行が更新
    6. [戻る] でメインへ

    押すボタン¶

    ボタン 動作
    [読込] 指定 KnwNo の現在値を画面に展開
    [更新] 編集内容を保存 → ナレッジ一覧の該当行を更新
    [削除] 確認ダイアログ後、該当ナレッジを削除
    [戻る] メインシートへ戻る

    ナレッジ一覧¶

    何ができるか¶

    登録済の全ナレッジを表形式で一覧表示します。KnwNo / FormatID / 主要フィールドの値が縦に並びます。

    標準操作手順¶

    1. メインシートの [ナレッジ一覧] ボタンを押す → ナレッジ一覧シートへ遷移
    2. 行をクリック → そのナレッジの詳細を確認可能（KnwNo 列をコピーしてナレッジ修正シートに貼り付ければ即編集できる）
    3. 列ヘッダーで Excel 標準のフィルタ・並び替えを使える
    4. [戻る] でメインへ

    押すボタン¶

    ボタン 動作
    [更新] 一覧を最新状態に再描画
    [戻る] メインシートへ戻る

    検索¶

    何ができるか¶

    キーワード + フォーマット指定でナレッジを検索し、スコア降順で結果を表示します。サムネイル画像（H 列）と Score 値（I 列）が結果と一緒に並びます。

    標準操作手順¶

    1. メインシートの [検索] ボタンを押す → 検索シートへ遷移
    2. 検索条件を入力:
      • FormatID: 検索対象フォーマット（空欄なら全フォーマット）
      • キーワード: 半角スペース区切りで複数指定可
      • 検索モード: AND / OR
      • TargetField: 検索対象フィールド（(全フィールド) で全文検索）
    3. [検索実行] ボタンを押す
    4. 14 行目以降にスコア降順で結果が並ぶ
    5. 結果行を選択 → 詳細プレビュー（任意） : Alt + F8 → Macro_ShowSearchResultPreview → [実行]

    押すボタン¶

    ボタン 動作
    [検索実行] 入力条件で検索 → 14 行目以降に結果展開
    [クリア] 検索条件・結果をクリア
    [戻る] メインシートへ戻る

    詳細プレビュー UserForm の前提条件

    セットアップガイド STEP 1 の [VBA プロジェクト オブジェクト モデルへのアクセスを信頼する] が ON である必要があります。検索のみなら OFF でも可。

    フォーマット設計¶

    何ができるか¶

    新規フォーマット（フィールド構成）を定義します。FormatID を決め、フィールド名・型・必須フラグ等を行で記述して登録します。

    標準操作手順¶

    1. メインシートの [フォーマット設計] ボタンを押す → フォーマット設計シートへ遷移
    2. FormatID に新規 ID を入力（例: MY-RECIPE）
    3. フィールド定義行に以下を記入:
      • フィールド名（例: 料理名, 材料, 調理時間）
      • 型（text, multiline, date, number, image, select 等）
      • 必須フラグ（Y / N）
      • 選択肢（型が select の場合のみ、カンマ区切り）
    4. [登録] ボタンを押す → フォーマット一覧に追加され、ナレッジ登録時に選べるようになる

    押すボタン¶

    ボタン 動作
    [登録] 入力したフォーマット定義を保存
    [プレビュー] フォーマットプレビューシートで入力フォーム見本を表示
    [クリア] フィールド定義行をクリア
    [戻る] メインシートへ戻る

    フォーマット一覧¶

    何ができるか¶

    登録済フォーマットの一覧を表形式で表示します。

    標準操作手順¶

    1. メインシートの [フォーマット一覧] ボタンを押す → フォーマット一覧シートへ遷移
    2. 各行に FormatID、フィールド数、登録日時等が並ぶ
    3. 行を選択 → そのフォーマットのフィールド構成を確認可能
    4. [戻る] でメインへ

    押すボタン¶

    ボタン 動作
    [更新] 一覧を最新状態に再描画
    [戻る] メインシートへ戻る

    フォーマットプレビュー¶

    何ができるか¶

    選択中フォーマットの入力フォーム見本を表示します。実際にナレッジ登録するとどんな画面になるかを事前に確認できます。

    標準操作手順¶

    1. メインシートの [フォーマットプレビュー] ボタンを押す → フォーマットプレビューシートへ遷移
    2. FormatID セルに対象 ID を入力 → [表示] ボタンを押す
    3. 画面にそのフォーマットの入力フォームが描画される（実入力はしない、見本のみ）
    4. [戻る] でメインへ

    押すボタン¶

    ボタン 動作
    [表示] 指定 FormatID のフォーム見本を描画
    [戻る] メインシートへ戻る

    既存データへのフィールド反映¶

    何ができるか¶

    フォーマット定義の変更（フィールド追加・名称変更等）を、既存の登録済ナレッジに対しても伝播させます。

    標準操作手順¶

    1. メインシートの [既存データへのフィールド反映] ボタンを押す → 該当シートへ遷移
    2. FormatID に対象フォーマット ID を入力
    3. [差分プレビュー] ボタンを押す → 既存ナレッジに対して何が追加・変更されるかを表示
    4. 内容を確認して [反映実行] ボタンを押す → 既存ナレッジに対する一括更新が実行
    5. 完了メッセージが出たら [戻る] でメインへ

    押すボタン¶

    ボタン 動作
    [差分プレビュー] 反映前後の差分を一覧表示（Dry-run）
    [反映実行] 差分を実適用
    [戻る] メインシートへ戻る

    実行前に必ずバックアップ

    一括更新は元に戻せません。実行前に .xlsm のコピーを別フォルダに保存してから操作してください。

    格納先設定¶

    何ができるか¶

    ナレッジテキスト・画像の保存先フォルダを指定します。

    標準操作手順¶

    1. メインシートの [格納先設定] ボタンを押す → 格納先設定シートへ遷移
    2. dataFolder に絶対パスを入力（例: C:\Users\<あなた>\Documents\knowledge\data\）
    3. 画像保存先（規約: <dataFolder の親>\kb_images\<KnwNo>.png）を確認
    4. [保存] ボタンを押す
    5. [戻る] でメインへ

    押すボタン¶

    ボタン 動作
    [保存] 指定パスを保存
    [フォルダ確認] 指定パスをエクスプローラで開く
    [戻る] メインシートへ戻る

    画像配置の規約

    dataFolder = C:\knowledge\data\ の場合、画像は C:\knowledge\kb_images\ に配置。ImagePath スタンザを使う場合は .txt 末尾に ImagePath=KN-2026-0420.png 等を書きます。

    設定¶

    何ができるか¶

    テストモード切替・ログ出力先・その他のシステム設定を行います。

    標準操作手順¶

    1. メインシートの [設定変更] ボタンを押す → 設定シートへ遷移
    2. 各項目を編集:
      • テストモード (C6): "TRUE" でテスト用、"FALSE" で本番モード
      • その他の項目（ログレベル等）
    3. [保存] ボタンを押す
    4. [戻る] でメインへ

    押すボタン¶

    ボタン 動作
    [保存] 設定値を保存
    [戻る] メインシートへ戻る

    ログ確認¶

    何ができるか¶

    実行ログを表形式で確認します。各操作の日時・モジュール名・関数名・メッセージ種別・メッセージ内容が記録されます。

    標準操作手順¶

    1. メインシートの [ログ] ボタンを押す → ログシートへ遷移
    2. ログシートの先頭行はヘッダー（日時 / モジュール名 / 関数名 / メッセージ種別 / メッセージ内容）
    3. 2 行目以降に新しい順でログが並ぶ
    4. Excel 標準のフィルタで「メッセージ種別 = ERROR」だけに絞ってトラブル原因を特定できる
    5. [戻る] でメインへ

    押すボタン¶

    ボタン 動作
    [ログクリア] 確認ダイアログ後、ログを全削除
    [戻る] メインシートへ戻る

    トラブルシューティング¶

    Q1. ボタンを押しても何も起きない¶

    VBA がコンパイル失敗している可能性。Alt + F11 で VBE を開き、[デバッグ] → [VBAProject のコンパイル] を実行してエラー内容を確認してください。多くは セットアップガイド Q5 のモジュールインポート漏れです。

    Q2. 検索結果の H 列に画像が表示されない¶

    kb_images/ フォルダが .xlsm と同じ階層にあるか確認してください。規約は <dataFolder の親>/kb_images/<KnwNo>.png。

    Q3. 「インデックスが有効範囲にありません」が出る¶

    シートが不足している可能性。メインの [▶初回セットアップ] をもう一度押してください。

    Q4. ボタンの位置が崩れた¶

    [▶初回セットアップ] を再実行すると同名ボタンを削除→再配置するので、位置が直ります。[▶初回セットアップ] は何度押しても安全です。

    Q5. 「マクロが見つかりません」と出る¶

    ブックを「名前を付けて保存」でリネームした後にこのエラーが出る場合、OnAction 設定がブック名修飾で動的に紐づいているのが原因。再度 [▶初回セットアップ] を押してください。

    制約・既知の限界¶

    • VBA 子プロセス禁止（Shell/Run/WScript.Shell/Exec）— 職場 PC ポリシー (ADR-0002)
    • ChromaDB 等の外部 RAG 連携無効化 — Cowork 隔離 (ADR-0004)
    • クラスモジュール (.cls) 内の Public Const/Type/Declare/Static 禁止 (ADR-0027)
    • Excel 単体動作のため Web/モバイルでは利用不可
    • 同時編集なし（ファイルベース）
    • 検索性能はモジュール数 O(n) 線形（数千ナレッジまでは実用）

    詳細 ADR は GitHub を参照。
