    操作手順¶

    このページは、セットアップ完了後の 日常操作（検索・登録・編集・画像添付・ログ確認） を集約しています。インストールやセットアップマクロの実行手順は セットアップ を参照してください。

    ページ構成

    検索 → 詳細表示 → 登録 → 編集 → 画像添付 → ログ確認 → dataFolder 切替 → フォーマット管理 → ショートカット → 拡張テスト → トラブルシューティング、の順に並んでいます。目的の操作は左ナビゲーションまたは目次（右側）から直接ジャンプできます。

    前提

    セットアップ を完了し、SetupSheetsAndButtons 実行後の状態であること。メインシートに 12 個のボタンが並んでいる状態。

    1. ナレッジを検索する¶

    Step 1.1 検索シートを開く¶

    メインシートの ▶検索 ボタンをクリック → 検索シートに自動遷移します。

    Step 1.2 検索条件を入力¶

    検索シート上部の入力欄に検索条件を記入します。

    項目 入力例 説明
    FormatID（行 6） DEMO-MEMO 対象フォーマットの ID。空欄なら全フォーマット対象
    キーワード（行 7） メモ 障害 スペース区切りで複数指定可
    検索モード（行 8） AND または OR 複数キーワード時の論理
    TargetField（行 9） (全フィールド) または特定フィールド名 検索対象を限定する場合に指定
    作成日 From（行 10） 2026-04-20 任意。CreatedDate の下限
    作成日 To（行 11） 2026-05-04 任意。CreatedDate の上限

    Step 1.3 検索実行¶

    検索シートの ▶検索実行 ボタンをクリックします。

    数秒で 14 行目以降に スコア降順 で結果が並びます。

    列 内容
    A 通し番号（1 から）
    B ナレッジ番号 (KN-2026-XXXX)
    C フォーマット名
    D タイトル
    E 作成日
    F 更新日
    G 「▶ 詳細」ラベル（クリック対象）
    H サムネ画像 (60×40px)
    I スコア値（0.00 形式）

    Step 1.4 スコアの読み方¶

    スコアは次の式で計算されます。詳細は 仕様 §3 参照。

    スコア = 出現回数(全フィールド) + 出現回数(タイトル) × 3 + 出現回数(対象フィールド) × 2

    スコアが大きいほど検索条件にマッチしているナレッジです。スコア 0 のナレッジは結果に含まれません。

    2. 検索結果から詳細を表示する¶

    方法 A: ナレッジ表示シートで開く¶

    1. 検索結果の B 列（ナレッジ番号セル）をダブルクリック、または番号をコピー
    2. メインシートの ▶ナレッジ表示 ボタンをクリック → ナレッジ表示シートに遷移
    3. ナレッジ表示シートの C1 セルに番号を入力し ▶表示実行 をクリック
    4. 各フィールドの値が一覧表示され、画像があれば J4:N20 にプレビューされる

    方法 B: 詳細プレビュー UserForm を開く（image_ext rev1）¶

    1. 検索シートで結果行の任意セルを選択（行が 1 つに決まればよい）
    2. Alt + F8 → Macro_ShowSearchResultPreview → [実行]
    3. spec 駆動 UserForm が開く（タイトル / 本文 / 画像 / 各フィールド表示）

    前提条件

    [トラスト センター] → [マクロの設定] → [VBA プロジェクト オブジェクト モデルへのアクセスを信頼する] を ON にする必要あり（セットアップ §1.2 参照）。

    3. ナレッジを新規登録する¶

    Step 3.1 フォーマットを確認¶

    新規ナレッジを登録するには、その記載フォーマットが「フォーマット一覧」シートに登録済である必要があります。デモ環境では DEMO-MEMO が登録済です。新規フォーマットを追加したい場合は §5 フォーマット管理 参照。

    Step 3.2 ナレッジ登録シートを開く¶

    メインシートの ▶ナレッジ登録 ボタンをクリック。

    Step 3.3 メタ情報を入力¶

    ナレッジ登録シートの上部に次を入力します。

    • ナレッジ番号 (C1): KN-2026-XXXX 形式。重複しない値
    • FormatID (C2): 利用するフォーマットの ID（例: DEMO-MEMO）

    Step 3.4 フィールドを入力¶

    メタ情報入力後、▶フォーマット読込 ボタンをクリックすると、フォーマット定義に基づくフィールドが 4 行目以降に展開されます。各フィールドの C 列に値を入力します。

    Step 3.5 保存¶

    ▶保存 ボタンをクリックすると、<dataFolder>/<ナレッジ番号>.txt が Shift_JIS + CRLF で書き出されます。同名ファイル既存時は上書き確認のダイアログが出ます。

    4. ナレッジを編集・更新する¶

    Step 4.1 編集対象を読み込む¶

    1. メインシートの ▶ナレッジ登録 ボタンをクリック → ナレッジ登録シートへ
    2. C1 に編集したい既存ナレッジ番号を入力
    3. ▶読込 ボタンをクリック → 既存ファイルの内容が各フィールドにロードされる

    Step 4.2 値を編集して保存¶

    各フィールドの値を編集後、▶保存 をクリック。UpdatedDate が現在日時で自動更新されます（CreatedDate は維持）。

    5. ナレッジを削除する¶

    1. メインシートの ▶ナレッジ削除 ボタンをクリック → ナレッジ削除シートへ
    2. C1 にナレッジ番号を入力
    3. ▶削除実行 ボタンをクリック
    4. 確認ダイアログが出るので OK で確定

    <dataFolder>/<ナレッジ番号>.txt がファイルシステム上から削除されます（ゴミ箱には行きません）。サムネ画像 kb_images/<ナレッジ番号>.png は 削除されません（画像は別管理）。

    削除は元に戻せません

    削除前にバックアップを取ることを推奨します。

    6. 画像（サムネ）を添付する¶

    ナレッジには 1 件あたり 1 枚の画像を関連付けできます。

    規約 A: ナレッジ番号と同名（推奨）¶

    <dataFolder> の 親フォルダ に kb_images/ を作成し、<ナレッジ番号>.png を置きます。

    C:\Users\<あなた>\Documents\knowledge_demo\
├─ data\
│   ├─ KN-2026-0420.txt
│   └─ ...
├─ kb_images\
│   ├─ KN-2026-0420.png  ← この命名なら自動で紐づく
│   └─ ...
└─ knowledge_test_v2_demo_v6.xlsm

    サイズの目安は 60×40px（サムネ表示時）〜 任意（詳細表示時は J4:N20 にフィット）。PNG / JPG / GIF / BMP に対応。

    規約 B: ImagePath スタンザで明示指定¶

    .txt ファイルの末尾に次の行を追加すると、任意のパスを画像として指定できます。

    ImagePath=C:\path\to\image.png

    または kb_images/ 配下の任意名で:

    ImagePath=custom_diagram.png

    詳細は 仕様 §4.2 参照。

    画像が表示されない場合¶

    検索結果 H 列に画像が出ない場合は次を確認します。

    • kb_images/ フォルダが <dataFolder> の 親階層 にあるか
    • ファイル名が <ナレッジ番号>.png (拡張子小文字) と一致しているか
    • ImagePath スタンザを使っている場合、ファイルが実在するか

    7. ログを確認する¶

    7.1 操作ログを見る¶

    メインシートの ▶ログ表示 ボタンをクリック → ログシートに遷移。各操作（検索 / 登録 / 削除 / エラー）の履歴が新しい順で並んでいます。

    列 内容
    A 日時 (yyyy-mm-dd hh:mm:ss)
    B モジュール名
    C 関数名
    D 種別（INFO / WARN / ERROR）
    E メッセージ

    7.2 ログをクリアする¶

    ログシートの最下部 ボタン ▶ログクリア をクリック → 1 行目のヘッダーを除く全行を削除。

    ログは Workbook を開くたびに自動でクリアされる（デモ環境のみ）ため、操作トレースを取りたい場合は早めにエクスポート/コピーしてください。

    8. dataFolder を切り替える¶

    複数のナレッジ集を運用したい場合、設定シートで dataFolder を切り替えます。

    Step 8.1 設定シートを開く¶

    メインシートの ▶設定 ボタンをクリック。

    Step 8.2 フォルダパスを入力¶

    設定シートの行 3 列 C（dataFolder）に新しいフォルダパスを直接入力するか、▶フォルダ選択 ボタンで GUI 選択します。例:

    C:\Users\<あなた>\Documents\knowledge_workA\data

    Step 8.3 反映確認¶

    設定変更後、検索 / 登録 / 表示などの操作が新しい dataFolder を参照します。kb_images/ も自動で <新しい dataFolder の親>/kb_images/ を参照します。

    9. フォーマットを管理する¶

    9.1 フォーマット一覧を開く¶

    メインシートの ▶フォーマット一覧 ボタンをクリック。フォーマット一覧シートに遷移します。

    9.2 新規フォーマットを追加¶

    1. 行 3 以降の空行を 1 行確保
    2. 列 A: FormatID（一意）
    3. 列 B: フォーマット名
    4. 列 C 以降: フィールド定義 DSL（FieldNo=1 / FieldName=タイトル / Type=Text / Required=Y 等）

    9.3 既存フォーマットを編集¶

    定義変更時、既存ナレッジファイル（.txt）のスキーマも自動移行されます（clsFieldMigrator が処理）。詳細は 仕様 §4 参照。

    11. 拡張テストの実行（任意 / 開発者向け）¶

    dev 専用テストモジュール（modTestRunnerExt.bas）を別途インポートしている場合のみ実行できます。配布物には含まれません。

    Application.Run "TestRunner_RunAll"      ' 既存 82 件
Application.Run "TestRunnerExt_RunAll"   ' 拡張 7 件

    合計 PASS 89 / SKIP 4 / FAIL 0 が完了条件です。

    12. 操作時のトラブルシューティング¶

    Q1. 検索結果の H 列に画像が表示されない¶

    §6 画像（サムネ）を添付する の規約と確認手順を参照。最も多い原因は kb_images/ フォルダが dataFolder と同階層に置かれているケース（親階層 が正しい）。

    Q2. 検索しても 0 件しかヒットしない¶

    • FormatID を間違えていないか（空欄にして全フォーマット対象で再実行）
    • キーワードが半角スペース区切りになっているか（全角空白は区切りにならない）
    • 検索モードが AND で全キーワードがそのファイルに無い場合は 0 件。OR で再実行
    • 作成日 From / To で対象期間を絞りすぎていないか（空欄にすると無制限）

    Q3. UserForm が開かない / マクロが実行できない¶

    セットアップ §1.2 で「VBA プロジェクト オブジェクト モデルへのアクセスを信頼する」を ON にしているか確認。OFF の状態では Forms.Add が失敗します。

    Q4. ナレッジ登録時に「フォーマットが見つかりません」と出る¶

    入力した FormatID がフォーマット一覧シートに存在しない可能性。フォーマット一覧で FormatID を確認してから再入力してください。

    Q5. dataFolder を切り替えたら検索が空になる¶

    新しい dataFolder に .txt ファイルが入っているか確認してください。空フォルダを指定すると検索結果はゼロになります。

    Q6. 画像が縦横比おかしく表示される¶

    Shapes.AddPicture の LockAspectRatio 引数で MsoTriState の解釈差で起こる場合があります。v6 で修正済（modImageRender 参照）。古いバージョンを使っている場合は v6 ベースに更新してください。

    13. 既知の制約・運用上の注意¶

    制約¶

    • Macro_ShowSearchResultPreview は Windows 版 Excel 限定 — Forms.* ProgID が必要なため Mac 版では実行できません（セットアップ §0）
    • 検索は単純スコアリング — タイトル ×3 + 対象フィールド ×2 + 出現回数の単純加算で、ベクトル検索（意味検索）はできません。同義語や類義語による絞り込みが必要な場合は別系統の検索基盤への移行が必要
    • 件数規模の上限の目安 — 数百〜千件程度のテキストファイルなら実用速度で動作します。数千件以上では再評価が必要です

    よくある運用ミス（チェックリスト）¶

    • 全角スペースで区切ってしまい、複数キーワードが認識されていない
    • dataFolder を変更したのに対応する kb_images/ を dataFolder の親 に置き忘れている
    • 新規フォーマットを追加したが、対応するナレッジファイルの FormatID= を未記入
