# 困ったとき

症状別の対処法をまとめています。当てはまる症状を選んで、上から順に試してください。

---

## ブックを開いた瞬間にコンパイル エラーが出る

**症状**: `登録修正.xlsm` / `検索.xlsm` / `管理.xlsm` のいずれかを開くと、Excel 起動と同時に **「Microsoft Visual Basic for Applications コンパイル エラー」** ダイアログが表示される。

**対処**:

1. ダイアログの **[OK]** を押し、Excel を一度閉じる
2. Excel の確認メッセージで **[保存しない]** を選ぶ
3. [インストール](install.md) の STEP 7 に戻り、該当ブックの bat（`install_register.bat` / `install_search.bat` / `install_admin.bat`）を再度ダブルクリックして実行
4. `[DONE] Install + setup succeeded.` を確認してから、Excel でブックを開き直す

それでも同じエラーが出る場合は、VBA モジュールファイル（`.bas` / `.cls`）の保存形式を確認してください。**ANSI（Shift-JIS）+ CRLF** で保存されている必要があります。

---

## ボタンを押しても何も起きない

**症状**: 「新規登録」「検索」「保存」等のボタンをクリックしても、画面が反応しない。エラーも何も出ない。

**対処**:

1. Excel 上部に **「セキュリティの警告 マクロが無効にされました」** の黄色いバーが出ていないか確認 → 出ていれば **[コンテンツの有効化]** をクリック
2. 該当ブックを閉じ、[インストール](install.md) の STEP 7 で対応する bat を再実行
3. それでも反応しない場合は、`installer\vba_modules\` 内のモジュールファイルがすべて揃っているか確認

---

## 「フォーマット」 ドロップダウンが空

**症状**: 「ナレッジ登録」画面や「検索」画面でフォーマット ドロップダウンを開いても、選択肢が何も出ない。

**対処**:

1. `format\` フォルダの中身を確認 → `.txt` ファイルが 0 件なら、フォーマット定義が登録されていません
2. `管理.xlsm` を開き、**「フォーマット一覧」シート（M-02）** → **[新規作成]** ボタンから少なくとも 1 つフォーマットを作成（[使い方 3.2](usage.md#32) を参照）
3. フォーマット作成後、`登録修正.xlsm` / `検索.xlsm` をいったん閉じて開き直す

---

## 検索結果が出ない

**症状**: 「検索」画面で **[検索]** ボタンを押しても、結果一覧が空のまま。

**対処**:

1. **[フォーマット]** が「すべて」になっているか、登録済みフォーマットになっているか確認 → 登録済みナレッジが存在しないフォーマットを選んでいると結果は空になります
2. **[キーワード]** を空欄にして、もう一度 **[検索]** をクリック → これでも空なら `data\` フォルダにナレッジ（`.txt`）が 1 件もありません
3. `data\` の中身を確認し、空であれば `登録修正.xlsm` から最低 1 件ナレッジを登録してください
4. キーワードが入っている状態で結果が出ない場合は、より短い・一般的なキーワードに変えて試す

---

## 「新規登録」の入力ウィンドウの幅を変えたい / 広げても変わらない

**症状**: 「ナレッジ登録」画面の **[新規登録]** で開く入力ウィンドウの横幅を変えたい。値を変えても幅が変わらない。

**対処**:

1. メモ帳で `C:\KnowledgeMgr\ui\検索\M-05.txt` を開く
2. `[USERFORM]` の `formWidth` の数値（ピクセル）を変更する（例: `formWidth=486` → `formWidth=620`）
3. **文字コードを「ANSI」(Shift_JIS)** にして、同じ場所へ上書き保存する
4. 開いているブックをいったん閉じて、開き直す

> 幅が変わらないときは、編集したファイルが `ui\検索\` フォルダの `M-05.txt` であること、保存時の文字コードが **ANSI** になっていることを確認してください。

---

## 「格納先を選ぶ」を押してもダイアログが出ない

**症状**: `管理.xlsm` の「格納先設定」画面で **[格納先を選ぶ]** ボタンを押しても、フォルダ選択ダイアログが出ない。

**対処**:

1. 行の左端の **チェックボックス** に、変更したいフォルダのチェックを入れる（最低 1 つ必要）
2. もう一度 **[格納先を選ぶ]** をクリック

チェックを 1 つも入れていない状態ではダイアログは開きません（誤操作防止のため）。

---

## ナレッジ修正が上書きされない / 「ナレッジが見つかりません」

**症状**: 「ナレッジ修正」画面でナレッジ番号を入れて **[読込]** を押すと、エラーや「見つかりません」が出る。

**対処**:

1. ナレッジ番号の形式を確認 → 正しい形式は `<フォーマット ID>-NNNN`（例: `FAULT-0001`）です。半角ハイフン 1 つで区切ります
2. `data\` フォルダ内に該当する `.txt` ファイルがあるか確認 → ファイル名がナレッジ番号と完全一致している必要があります
3. 格納先フォルダ設定が正しいか、`管理.xlsm` の「格納先設定」画面で確認
4. ナレッジ番号の英大文字・小文字が一致しているか確認（例: `fault-0001` ではなく `FAULT-0001`）

---

## 設定ファイルの保存先を変更したら、エラーメッセージが表示される

**症状**: 設定ファイル（`管理_config.txt` など）で保存先フォルダ（`data_dir` / `format_dir` / `ui_dir` / `backup_dir`）を別の場所に書き換えたあと、ブックを開くと **「ファイルが見つかりません」「画面定義ファイルが見つかりません」** といったメッセージが出る、または以前のデータが表示されない。

**対処**:

1. 設定ファイルに書いた各フォルダ（`data_dir` / `format_dir` / `ui_dir` / `backup_dir`）が、**実際に存在するか**を確認します。なければ作成してください
2. 元のフォルダに入っていた `.txt`（ナレッジ・フォーマット・**画面定義**）を、新しいフォルダへ**移動またはコピー**します。とくに `ui_dir`（画面定義）の中身が無いと、画面を開くときにメッセージが出ます
3. パスは**半角**で書き、**末尾は `\`（円マーク）** で終えます（例: `data_dir=D:\業務データ\KnowledgeMgr\data\`）
4. 3 つの設定ファイルすべてを同じ保存先に揃え、文字コードを **ANSI**（Shift-JIS）にして上書き保存します
5. ブックをいったん閉じて、開き直します

> 保存場所をまとめて変える手順は [インストール手順の 4.5.6](install.md) にもまとめています。フォルダを先に作ってから設定ファイルを書き換えるのが安全です。

## 期待していたシートが見えない / タブが少ない

**症状**: ブックを開いてもタブが 1?2 個しか見えない、本来あるはずの「フォーマット一覧」「格納先設定」等のタブが表示されない。

**対処**:

1. シートタブ部分で右クリック → **[再表示]** を選ぶ
2. 表示されたシート一覧から見たいシートを選び、**[OK]** をクリック

knowledgevba は「業務に必要なシートだけを見せる」設計のため、起動時に他のシートは非表示にしています。手動で再表示すれば一時的に表示できます。

## ナレッジ番号が飛ぶ（連番にならない）

**症状**: 続けて登録したナレッジの番号が `FAULT-0001` → `FAULT-0003` のように飛んでいる。

**対処**:

これは正常動作です。複数の利用者が同じ瞬間に登録した場合や、登録途中でキャンセルされた場合に番号がスキップされることがあります。データの欠落ではないので、そのまま使用して問題ありません。

---

## bat 実行中に `[ERROR] VBA プロジェクトへのアクセスが拒否されました`

**症状**: install bat を実行すると、上記メッセージで停止する。

**対処**:

1. [インストール STEP 1](install.md#step-1) の「Excel の事前設定」を実施
2. Excel の **[ファイル]** → **[オプション]** → **[トラスト センター]** → **[トラスト センターの設定]** → **[マクロの設定]** で **[VBA プロジェクト オブジェクト モデルへのアクセスを信頼する]** にチェック
3. Excel をすべて閉じてから、bat を再実行

---

## bat 実行中に `[ERROR] EXCEL.EXE が kill 後も残存しています`

**症状**: install bat が Excel プロセスの強制終了に失敗してエラー停止する。

**対処**:

1. すべての Excel ウィンドウを手動で閉じる
2. タスクマネージャー（`Ctrl+Shift+Esc`）を開き、**[詳細]** タブで `EXCEL.EXE` が残っていないか確認 → 残っていれば右クリックで **[タスクの終了]**
3. 1?2 分待ってから bat を再実行

---

## bat 実行中に `[ERROR] vba_modules\xxx フォルダが見つかりません`

**症状**: install bat が、`common` / `register` / `search` / `admin` のいずれかのフォルダがないと停止する。

**対処**:

1. `C:\KnowledgeMgr\installer\vba_modules\` を開く
2. 不足しているサブフォルダを作成
3. [インストール STEP 6](install.md) の指示どおりに、対応する `.bas` / `.cls` ファイルをサブフォルダに配置
4. bat を再実行

---

## それでも解決しないとき

- すべての Excel を閉じ、`installer\` フォルダ内の 3 つの bat を最初から順番に再実行してみてください
- フォルダ構成は [インストール](install.md) の各 STEP どおりになっているか確認し
