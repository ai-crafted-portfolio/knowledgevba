# 設定値の変更

ナレッジ管理ツール v2.1 の動作は、ブックごとの動作設定ファイルで調整します。このページでは、利用者が変更する設定値とその変更方法を説明します。

## 1. 動作設定ファイル

設定は、ブックごとに 1 つの動作設定ファイルにまとまっています。

- `登録修正_config.txt`
- `検索_config.txt`
- `管理_config.txt`

3 ファイルとも、対応するブックと同じフォルダ（既定では `C:\KnowledgeMgr\`）に置きます。中身は `[CONFIG]` セクションに `key=value` を並べたテキストです。

```
[CONFIG]
debugLevel=INFO
logRotationRows=10000
data_dir=C:\KnowledgeMgr\data\
format_dir=C:\KnowledgeMgr\formats\
ui_dir=C:\KnowledgeMgr\ui\
backup_dir=C:\KnowledgeMgr\data\backup\
uiSchemaFailMode=safeDefault
systemSheetVisibility=Hidden
autoReloadOnStartup=TRUE
migrateBackupEnabled=TRUE
```

### 変更のしかた

設定ファイルはテキストエディタ（メモ帳など）で直接編集するのが基本です。

1. 編集の前にファイルのコピーを取っておきます。
2. テキストエディタで開いて値を書き換えます。
3. **文字コードは Shift_JIS（ANSI）、改行は CRLF** で上書き保存します。
4. 設定はブックの起動時に読み込まれます。変更を反映するには、対象のブックを開き直します。

格納先フォルダと一部の動作設定は、ブックの画面からも変更できます（後述）。

!!! note "設定ファイルが壊れているとき"
    設定ファイルが読み込めない場合でも、ツールは既定値で起動します。個々の項目に書き間違いがあった場合は、その項目だけ既定値が使われ、起動は続行されます。

## 2. 格納先フォルダ

ナレッジ・フォーマット・画面定義・バックアップを置くフォルダを指定します。いずれも絶対パスで、末尾に `\` を付けます。

| 設定キー | 内容 |
|---|---|
| `data_dir` | ナレッジ本体（`<ナレッジ番号>.txt`）の置き場所 |
| `format_dir` | フォーマット定義（`<フォーマット ID>.txt`）の置き場所 |
| `ui_dir` | 画面レイアウト定義（`<ブック>\<画面>.txt`）の置き場所 |
| `backup_dir` | フォーマット変更時などに取得するバックアップの置き場所 |

格納先フォルダは、管理.xlsm の **M-10 格納先設定**画面からも変更できます。フォルダを別の場所へ移したときは、ファイルを移動したうえでこの 4 つのパスを合わせてください。

## 3. ログの詳細度（debugLevel）

`debugLevel` は、LOG シートに残す操作ログの詳しさを決めます。値は次の 6 段階です。上にいくほど記録量が少なく、下にいくほど詳しくなります。

| 値 | 記録される内容 |
|---|---|
| `OFF` | 記録しない |
| `ERROR` | エラーのみ |
| `WARN` | エラーと警告 |
| `INFO` | 上記に加えて通常の操作 |
| `DEBUG` | 上記に加えて開発・調査向けの詳細 |
| `TRACE` | 最も詳しい記録 |

設定ファイルに `debugLevel` の記載が無い場合の既定は `ERROR`（最小限）です。通常の運用では `INFO` を推奨します。不具合を調べたいときだけ一時的に `DEBUG` や `TRACE` に上げ、調査が終わったら戻すと、LOG シートが余計に大きくなりません。

`debugLevel` は、管理.xlsm の **M-11 設定**画面からも変更できます。

関連する設定として、`logRotationRows` は LOG シートに保持するログ行数の上限です（既定 10000 行）。上限に達すると古い行から自動で削除されます。

## 4. その他の動作設定

| 設定キー | 内容 | 既定値 |
|---|---|---|
| `uiSchemaFailMode` | 画面レイアウト定義に不備があったときの挙動。`safeDefault` はその画面だけエラー表示にして他の画面は通常起動、`abort` は 1 件でも不備があれば全画面の起動を中止 | `safeDefault` |
| `systemSheetVisibility` | LOG シートなど管理用シートの表示状態。`Hidden`／`Visible`／`VeryHidden` | `Hidden` |
| `autoReloadOnStartup` | 起動時にフォーマット定義・画面定義を読み直すかどうか | `TRUE` |
| `migrateBackupEnabled` | フォーマット変更を既存ナレッジへ反映する際にバックアップを取るかどうか。安全のため `TRUE` のままを推奨 | `TRUE` |

## 5. ファイル形式設定（M-13）

出力するファイルの形式に関する設定は、管理.xlsm の **M-13 ファイル形式設定**画面でまとめて行います。M-13 には文字コード・改行・テンプレートなどに関する 15 項目があり、画面上で値を選んで設定します。

M-13 の画面には、設定の不整合などを知らせるエラー表示帯（4 列の表）が連動して表示されます。

## 6. 画面の見た目を変えたいとき

ボタンの位置・ラベル・色、入力枠の大きさ、列幅といった画面の見た目は、動作設定ファイルではなく画面レイアウト定義ファイル（`ui\<ブック>\<画面>.txt`）で決まります。変更方法は[画面のカスタマイズ](screen_customization_guide_v2.md)を参照してください。
