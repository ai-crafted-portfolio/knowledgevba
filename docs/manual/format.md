# フォーマット設計の流れ

型（フォーマット）を作って、検索・登録で使えるようにする手順です。

## 新規に作る／既存を直す

### 🆕 新しく作る

- 管理ブックの一覧で『新規作成』を押す
- ID・名前・項目を決めて入力する
- 『保存』で新しい型として登録される

### ✏ 既存を直す

- 一覧で対象の行の『編集』を押す
- 項目の追加・削除・変更を行う
- 『保存』で上書きされる

!!! tip "直したあとに"
    既存の型を直したあとは、変更チェックで登録済みナレッジとの食い違いを点検しておくと安心です。

## 項目（フィールド）を組み立てる

項目は 1 行に 1 つずつ並べます。順番がそのまま登録画面の並び順になります。

- **項目名** — 入力欄に表示される名前（例：発生日、対応内容）
- **型** — 文字列／数値／日付／複数行 から選ぶ
- **必須・任意** — 必ず入れてほしい項目は『必須』にする

『複数行』を選ぶと、長い説明を改行つきで書ける入力欄になります。

!!! tip "設計のコツ"
    必須にしすぎると登録が面倒になります。本当に欠かせない項目だけを必須にするのがコツです。

## 保存して検索・登録から使う

1. **設計画面で『保存』を押す** — 型の定義が保存され、すぐに使える状態になります。
2. **検索ブックを開き直す** — 登録画面・検索画面のフォーマット選択に、新しい型が表示されます。
3. **型を選んで使い始める** — 登録ではその型で入力でき、検索ではその型で絞り込めます。

!!! tip "ブック間の連携"
    型・ナレッジ・設定はすべて同じ `C:\KnowledgeMgr\` 配下にあるため、管理ブックの変更が検索ブックにも反映されます。
