# セットアップ

ご利用前に一度だけ行う準備作業です。

## セットアップの全体の流れ

1. 保存先フォルダ（`C:\KnowledgeMgr\`）を作る
2. Excel を一度だけ事前設定する（マクロ関連）
3. ブックを開いたらマクロを有効にする
4. インストーラ STEP1 を実行する（ファイル生成）
5. インストーラ STEP2 を実行する（取り込み）
6. 画面の固定ボタンを配置する
7. ブックを開き直して動作を確認する

!!! tip "所要時間の目安"
    15〜30 分。①〜⑦ は最初の 1 回だけです。次回からはブックをダブルクリックするだけで使えます。

## STEP ①　保存先フォルダを作る

1. **エクスプローラーを開く** — Windows のエクスプローラー（フォルダの窓）を開きます。
2. **C ドライブの直下にフォルダを作る** — `C:\` の直下に「KnowledgeMgr」という名前のフォルダを新規作成します。
3. **配布物を中に置く** — 受け取ったインストーラ一式を、このフォルダの中に置きます。

!!! tip "補足"
    別の場所に作ることもできますが、特に理由がなければ `C:\KnowledgeMgr\` をおすすめします。

## STEP ②　Excel を一度だけ事前設定する

1. **Excel を起動し、オプションを開く** — 「ファイル」→「オプション」を選びます。
2. **トラスト センターの設定を開く** — 「トラスト センター」→「トラスト センターの設定」ボタンを押します。
3. **VBA プロジェクトへのアクセスを許可する** — 「マクロの設定」で『VBA プロジェクト オブジェクト モデルへのアクセスを信頼する』にチェックを入れます。
4. **OK で閉じ、Excel をすべて閉じる** — 設定後はいったん Excel をすべて終了します。

!!! warning "なぜ必要？"
    この設定をしないと、後の取り込み（STEP⑤）で『アクセスが拒否されました』と表示され先に進めません。

## STEP ③　ブックのマクロを有効にする

- 本ツールはマクロ（自動処理）で動くため、マクロを有効にする必要があります。
- ブックを初めて開くと、画面上部に黄色い『セキュリティの警告 マクロが無効にされました』のバーが出ます。
- そのバーの『コンテンツの有効化』ボタンをクリックします。
- 一度有効にすると、次回以降は同じブックでは警告が出にくくなります。

!!! tip "よくあるつまずき"
    ボタンを押しても画面が反応しないときは、まずこの黄色いバーが残っていないかを確認してください。

## STEP ④　インストーラ（前半）を実行する

1. **STEP1 のスクリプトを用意する** — 配布されたファイル生成スクリプト（kvba_1_files）を保存先フォルダに置きます。
2. **ダブルクリックで実行する** — 付属のバッチ（.bat）をダブルクリックすると実行できます。
3. **確認ダイアログ（UAC）に応答する** — 「このアプリがデバイスに変更を加えることを許可しますか？」が出たら『はい』を選びます。
4. **完了メッセージを確認する** — 『STEP 1/2 complete: files generated.』と表示されれば成功です。

!!! tip "このステップで起きること"
    この前半では、ブックの土台や部品ファイルがフォルダに用意されるだけです。Excel への取り込みは後半（STEP⑤）で行います。

## STEP ⑤　インストーラ（後半）を実行する

1. **STEP2 のスクリプトを実行する** — 取り込みスクリプト（kvba_2_import）を、付属バッチのダブルクリックで実行します。
2. **確認ダイアログ（UAC）に応答する** — 再び確認が出たら『はい』を選びます。Excel が自動で開閉することがあります。
3. **完了メッセージを確認する** — 『取り込みに成功しました』の表示と、検索ブック・管理ブックが用意されたことを確認します。

!!! tip "うまくいかないとき"
    前半が成功し後半で失敗した場合は『ファイル生成は成功／取り込みで失敗』と切り分けられます。STEP② の設定漏れがないか確認してください。

## STEP ⑥　画面の固定ボタンを配置する

- 各画面に常に 1 つだけ置く固定ボタン（検索・登録・保存・クリアなど）は、利用者が手作業で配置します。
- 一覧の各行に並ぶ『編集』『削除』などのボタンは、ツールが自動で作るので手作業は不要です。

1. **『開発』タブを表示する** — リボンに無ければ「オプション」→「リボンのユーザー設定」で『開発』にチェック。
2. **フォームコントロールのボタンを選ぶ** — 「開発」→「挿入」→『フォームコントロール』のボタンを選びます（ActiveX は使いません）。
3. **ドラッグして描き、マクロを割り当てる** — 画面上でドラッグして描き、表示されたダイアログで対応するマクロ名を選び OK。

!!! tip "配置の目安"
    配置するボタンの位置・名前・割り当てマクロは、各画面のレイアウト定義ファイルに記載されています。

## STEP ⑦　動作を確認する

- 検索ブック・管理ブックをいったん閉じ、もう一度ダブルクリックで開きます。
- 開いたときに、それぞれの最初の画面（検索ブックは検索画面、管理ブックはフォーマット一覧）が表示されることを確認します。
- 下のシートタブをクリックして、各画面に切り替わることを確認します。
- 配置したボタンを押して、想定どおりに動くことを確認します。
- ログ画面に起動の記録が残っていれば、セットアップ完了です。

!!! tip "完了！"
    うまく動かないときは、[困ったとき](faq.md) を順に確認してください。
