# 操作マニュアル

このページでは、ナレッジ管理ツールの各シートで「何ができるか」「どのボタンを押すと何が起きるか」を機能別に説明します。

セットアップは完了済の前提です。本ツールを開くと **メイン** シートが表示され、画面下部の 8 個のタスクボタンから目的の画面を選びます。

---

## メイン画面

メインシート下部に 8 個のタスク選択ボタンが並んでいます。各ボタンを押すと対応するシートだけが表示されます。

| ボタン | 遷移先 / 役割 |
|---|---|
| **> 初回セットアップ** | （セットアップ済の運用では使用しません） |
| **> 設定変更** | 設定 シート |
| **> フォーマット管理** | フォーマット一覧 シート |
| **> ナレッジ登録** | ナレッジ登録 シート |
| **> 検索・確認** | 検索 シート |
| **> ナレッジ修正** | ナレッジ修正 シート |
| **> ナレッジ削除** | ナレッジ一覧 シート |
| **> 既存データ反映** | 既存データへのフィールド反映 シート |

---

## 1. ナレッジ登録

新規のナレッジ（ノウハウ・トラブル対応メモ・手順書等）を登録するシートです。

**操作手順**

1. メイン → **> ナレッジ登録** ボタンをクリック
2. ナレッジ登録シートが表示される
3. 上部の **FormatID** 欄でフォーマットを選択（フォーマット未作成なら「フォーマット管理」で先に定義）
4. フォーマットに従って各フィールドを入力
5. **> 保存** ボタンをクリックすると、格納先設定で指定したフォルダにテキストファイルとして保存される
6. 続けて入力する場合は **> クリア** ボタンで入力欄を初期化

**主なボタン**

- **> 保存** … 入力中のフィールドをファイルに書き出す
- **> クリア** … 入力欄を空にする

---

## 2. ナレッジ修正

既存ナレッジを呼び出して内容を上書きするシートです。

**操作手順**

1. メイン → **> ナレッジ修正** ボタンをクリック
2. KnwNo（ナレッジ番号）を上部の入力欄に入れる
3. **> 読込** ボタンをクリックすると、各フィールドに既存値が展開される
4. 必要箇所を編集
5. **> 上書保存** ボタンで反映

**主なボタン**

- **> 読込** … 指定 KnwNo の既存ナレッジを画面に展開
- **> 上書保存** … 編集内容で既存ファイルを上書き

---

## 3. ナレッジ一覧

登録済ナレッジを一覧表示し、削除もここから行うシートです。

**操作手順**

1. メイン → **> ナレッジ削除** ボタンをクリック（一覧シートが開く）
2. **> リロード** ボタンで最新の登録状況を再読み込み
3. 削除したい行を選択
4. **> 選択行を削除** ボタンをクリック
5. 確認ダイアログで OK → ファイルが削除される

**主なボタン**

- **> リロード** … 一覧を再構築
- **> 選択行を削除** … 選択中の行に紐づくファイルを削除

---

## 4. 検索

登録されたナレッジを横断検索するシートです。スコアリング順に結果が並びます。

**操作手順**

1. メイン → **> 検索・確認** ボタンをクリック
2. **キーワード** 欄に検索語を入力（複数語はスペース区切り）
3. **モード** で AND / OR を選択
4. **TargetField** で検索範囲（全フィールド / タイトルのみ / 本文のみ等）を指定
5. **> 検索** ボタンをクリックすると結果が下方の一覧に表示される
6. 詳細を見たい行を選択 → **> 詳細** ボタンで「ナレッジ表示」シートに遷移
7. 直接 KnwNo がわかっている場合は、上部欄に番号を入れて **> 表示** ボタン

**主なボタン**

- **> 検索** … キーワード検索を実行
- **> 表示** … 直接 KnwNo 指定で開く
- **> クリア** … 入力条件と結果をクリア
- **> 詳細** … 選択行のナレッジを表示画面で開く

---

## 5. ナレッジ表示

検索結果から開いた個別ナレッジの詳細表示シートです。本文と関連画像（サムネ・詳細画像）が表示されます。

**主なボタン**

- **< 検索に戻る** … 検索シートへ戻る
- **> 修正に遷移** … 表示中のナレッジを修正シートに引き継いで開く

---

## 6. フォーマット設計

ナレッジのフィールド構造を定義する画面です。フォーマット ID ごとにフィールド名・型を宣言します。

**操作手順**

1. メイン → **> フォーマット管理** → 一覧シートで **> 新規作成** または既存行を選んで **> 選択行を編集**
2. フォーマット設計シートが開く
3. フォーマット ID と各フィールドを上から順に定義
4. **> 保存** ボタンで保存
5. **> プレビュー** ボタンで設計内容の確認画面を表示
6. **< 一覧に戻る** で一覧シートへ

**主なボタン**

- **> 読込** … 既存フォーマットを読み込む
- **> 保存** … 設計内容を保存
- **> プレビュー** … プレビュー画面を開く
- **< 一覧に戻る** … フォーマット一覧へ

---

## 7. フォーマット一覧

定義済フォーマットを一覧表示するシートです。

**主なボタン**

- **> 新規作成** … 新規フォーマット設計画面を開く
- **> 選択行を編集** … 選択行のフォーマットを設計画面で開く
- **> プレビュー** … 選択行のプレビュー表示

---

## 8. フォーマットプレビュー

設計中または既存フォーマットのフィールド構成を視覚的に確認する画面です。**フォーマット設計** または **フォーマット一覧** から **> プレビュー** ボタンで開きます。

---

## 9. 既存データへのフィールド反映

フォーマットを変更したあとに、過去ナレッジへ新フィールドを追加・写像するシートです。

**操作手順**

1. メイン → **> 既存データ反映** ボタンをクリック
2. シートで対象フォーマット ID と反映ルールを指定
3. **> 反映実行** ボタンをクリック
4. 結果ログが出力される

**主なボタン**

- **> 反映実行** … 対象ナレッジに新フィールドを書き込む

---

## 10. 格納先設定

ドキュメントタイプごとに保存先フォルダを設定するシートです。「共有フォルダ」「BOX」等の保存先タイプとパスを定義します。

**操作手順**

1. メイン → **> 設定変更** → 設定シートまたは格納先設定シートに遷移
2. 表に DocType / 保存先タイプ / パス を 1 行ずつ書き込む
3. シートを切り替えると即時保存される（ボタンなし）

---

## 11. 設定

データフォルダの基準パス、ログレベル等を設定するシートです。

**主なボタン**

- **> リセット** … ログをクリアする

---

## 12. ログ確認

ログは設定シートの下部または専用ログシートに直近の実行結果が時系列で記録されます。エラーや警告はここで確認できます。

ログは **> リセット** ボタン（設定シート）で初期化できます。
