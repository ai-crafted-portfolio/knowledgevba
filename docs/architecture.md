# 全体アーキテクチャ

このページは、ナレッジ管理ツール v2.1 が「どんな部品から成り、どう動くか」を説明します。具体的な値（画面一覧・データ書式など）は[仕様](spec.md)を参照してください。

## 1. 全体構成

ツールは、3 つの Excel ブックと、ブックの外側に置かれたデータフォルダで構成されます。

```
C:\KnowledgeMgr\
  ├ 登録修正.xlsm        … ナレッジの登録・修正
  ├ 検索.xlsm            … ナレッジの一覧・検索・閲覧
  ├ 管理.xlsm            … フォーマット設計・設定・操作ログ
  │
  ├ 登録修正_config.txt  … 各ブックの動作設定（起動時に読み込む）
  ├ 検索_config.txt
  ├ 管理_config.txt
  │
  ├ data\                … ナレッジ本体（1 件 1 ファイル）
  ├ formats\             … フォーマット定義
  └ ui\                  … 画面レイアウト定義（ブック・画面ごと）
      ├ 登録修正\
      ├ 検索\
      └ 管理\
```

3 つのブックは互いを直接開いたり参照したりしません。共有が必要な情報（フォーマット定義やナレッジ本体）は、すべて `data\` / `formats\` のファイルを介してやり取りします。

## 2. 3 ブックに分けた理由

旧バージョンは 1 つのブックにすべての画面を詰め込んでいましたが、v2.1 では役割ごとに 3 ブックへ分割しました。

- **利用者が必要な機能だけを開ける** — 検索しかしない人は検索.xlsm だけを開けばよく、フォーマット設計や設定画面に触れる必要がありません。
- **書き込みの責任範囲がはっきりする** — ナレッジ本体への書き込みは登録修正.xlsm だけ、フォーマット定義への書き込みは管理.xlsm だけ、と決めています。これにより、閲覧目的のブックが誤ってデータを変更する事故を防ぎます。
- **画面数が分散して見通しが良くなる** — 13 画面が 1 ブックに並ぶ代わりに、登録修正は 2 画面、検索は 3 画面、管理は 8 画面に分かれます。

## 3. スタンザ駆動のしくみ

本ツールの大きな特徴は、**画面の見た目もデータの形式も、Excel ブックの外側のテキストファイルで定義する**点です。これらのテキストは「スタンザ形式」（`[セクション]` 行と `key=value` 行の繰り返し）で書かれています。

ブックの外側で定義されるものは 4 種類あります。

| 定義 | 置き場所 | 役割 |
|---|---|---|
| 画面レイアウト定義 | `ui\<ブック>\<画面>.txt` | 各画面のセル幅・ラベル・入力枠・ボタン位置などの見た目 |
| フォーマット定義 | `formats\<フォーマット ID>.txt` | ナレッジの入力項目のひな型（項目名・型・必須かどうか） |
| 動作設定 | `<ブック名>_config.txt` | フォルダの場所・ログの詳細度などの設定値 |
| ナレッジ本体 | `data\<ナレッジ番号>.txt` | 登録された 1 件分の内容 |

ブックを開くと、VBA が画面レイアウト定義を読み込んでシート上に画面を組み立てます。入力フォームの項目はフォーマット定義から決まります。つまり、**画面に項目を足したり並べ替えたりする作業は、`.xlsm` のコードを書き換えずに、外部のテキストを編集するだけで行えます**。この編集方法は[画面のカスタマイズ](screen_customization_guide_v2.md)で説明しています。

## 4. 層構成

各ブックの中の VBA モジュールは、役割ごとに次の層に分かれています。呼び出しは上から下への一方向で、下の層が上の層を呼ぶことはありません。

| 層 | 役割 |
|---|---|
| エントリポイント層 | シート上のボタン操作や起動イベントを受け取る薄い受け口 |
| 画面層 | 各画面（M-02〜M-14）の組み立てと再描画 |
| ビジネスロジック層 | ナレッジ・フォーマット・検索・ログなど中心的な業務処理 |
| ユーティリティ層 | 外部ファイルの入出力、スタンザ解析、文字列・日付処理など |
| インストーラ層 | 空のブックに必要なシートを用意するセットアップ |
| 特殊モジュール | Excel がブックごとに自動で持つ ThisWorkbook（起動・終了イベント） |

各モジュールの一覧とソースコードは[ソースコード](source/index.md)にまとめています。

## 5. 外部ファイルの入出力を 1 か所に集約

外部のテキストファイルを読み書きするコードは、専用のモジュールにまとめています。画面層やビジネスロジック層が、ファイルを直接開いて読み書きすることはありません。

| モジュール | 担当するファイル |
|---|---|
| modConfigLoader | 動作設定ファイルの読み込み |
| modUILoader | 画面レイアウト定義の読み込みと画面への適用 |
| modFormatLoader | フォーマット定義の読み書き |
| modKnowledgeFileIO | ナレッジ本体の読み書き・バックアップ |
| modStanzaIO | スタンザ形式の共通的な解析・書き出し |

入出力を集約することで、文字コード（Shift_JIS）や改行（CRLF）の扱いを一定に保ち、書き込み権限（ナレッジは登録修正.xlsm のみ、フォーマットは管理.xlsm のみ）をこの層で確実に守れるようにしています。

## 6. ブック起動の流れ

各ブックを開くと、おおむね次の順序で初期化が進みます。

1. そのブックの動作設定ファイル（例：`登録修正_config.txt`）を読み込む。
2. 読み込んだ設定値をメモリ上に保持する。
3. 操作ログの記録を初期化する。
4. そのブックが担当する各画面について、画面レイアウト定義を読み込んでシートを組み立てる。
5. 起動時に表示する画面（登録修正は M-05、検索は M-08、管理は M-02）を開く。

起動後、シートタブで画面を切り替えると、その画面のレイアウト定義が読み直されて表示が更新されます。

## 7. 配布の考え方

完成した `.xlsm` ファイルそのものを配るのではなく、VBA モジュール一式を各 PC のブックに取り込み、セットアップ用のマクロを実行して使える状態にします。シート構成は、セットアップマクロが起動時に自動でそろえます。実際の手順は[導入手順](setup.md)を参照してください。
