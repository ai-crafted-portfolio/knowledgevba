# 画面のカスタマイズ

ナレッジ管理ツール v2.1 の画面の見た目は、Excel ブックの外側に置いたテキストファイルで定義されています。このページでは、その定義ファイルを編集して画面を調整する方法を説明します。

## 1. 画面レイアウトは外部ファイルで決まる

各画面（M-02?M-14）の見た目は、`ui_dir`（既定 `C:\KnowledgeMgr\ui\`）の下にある**画面レイアウト定義ファイル**で決まります。ファイルはブック・画面ごとに 1 つです。

```
C:\KnowledgeMgr\ui\
  ├ 登録修正\
  │   ├ M-05.txt
  │   └ M-06.txt
  ├ 検索\
  │   ├ M-07.txt
  │   ├ M-08.txt
  │   └ M-09.txt
  └ 管理\
      ├ M-02.txt ? M-04.txt
      └ M-10.txt ? M-14.txt
```

ブックを開くと、ツールがこの定義ファイルを読み込んでシート上に画面を組み立てます。**そのため、画面の見た目を変えるときは、Excel ブックの中の VBA コードを書き換えるのではなく、対応する画面レイアウト定義ファイルを編集します。**

定義ファイルは Shift_JIS、改行 CRLF のテキストです。編集の前に必ずコピーを取っておいてください。

## 2. 定義ファイルの書き方

画面レイアウト定義は「スタンザ形式」で書かれています。`[セクション名]` の行に続けて `key=value` の行を並べ、次の `[セクション名]` までが 1 つのセクションです。

```
[SHEET]
SheetName=M-05
SheetTitle=ナレッジ登録
TabColor=#FFCC00

[COLUMN]
ColumnWidth_A=4
ColumnWidth_C=60

[LABEL]
Cell=A6
Text=件名
Bold=TRUE

[BUTTON]
Cell=G2
Text=保存
OnClick=Btn_M05_Save
BackColor=#3366CC
ForeColor=#FFFFFF
```

使えるセクションは次の 11 種類です。

| セクション | 役割 |
|---|---|
| `[SHEET]` | シート名・タブ色・保護レベル（1 ファイルに 1 つ、必須） |
| `[COLUMN]` | 列幅 |
| `[ROW]` | 行の高さ |
| `[FONT]` | 既定のフォントとサイズ |
| `[LABEL]` | 固定の文字列（項目名など） |
| `[INPUT]` | 入力枠 |
| `[BUTTON]` | 画面に常時 1 つ置くボタン |
| `[BUTTON_TEMPLATE]` | 一覧の行ごとに増減するボタンのひな型 |
| `[HEADER]` | 画面上部の見出し帯 |
| `[NOTE]` | 説明文（画面下部にまとめて置く） |
| `[GRID]` | 一覧表 |

セクションを書く順番は自由です。ツールが読み込み時に適切な順序で画面へ反映します。

## 3. 1 つの画面の見た目を変える

特定の画面を調整するときは、その画面の定義ファイル（例：`ui\管理\M-02.txt`）を編集します。

### 例：ボタンの色を変える

`[BUTTON]` セクションの `BackColor`（背景色）・`ForeColor`（文字色）を `#RRGGBB` 形式で書き換えます。

```
[BUTTON]
Cell=G2
Text=保存
OnClick=Btn_M05_Save
BackColor=#548235      ← 緑にする
ForeColor=#FFFFFF
```

### 例：列の幅を変える

`[COLUMN]` セクションで `ColumnWidth_<列文字>` を書き換えます。値は Excel の列幅（おおよその文字数）です。

```
[COLUMN]
ColumnWidth_C=80       ← C 列を広げる
```

### 例：項目名（ラベル）を変える

`[LABEL]` セクションの `Text` を書き換えます。

```
[LABEL]
Cell=A6
Text=タイトル           ← 「件名」から変更
Bold=TRUE
```

### 入力枠の調整

`[INPUT]` セクションの `Cell` で入力枠の位置や範囲（`C6` のような単一セル、または `C7:E20` のような範囲）を、`InputType` で入力の種類（1 行テキスト・複数行・数値・日付・選択肢など）を指定します。入力内容の想定文字数に合わせて枠の大きさを調整してください。

## 4. ボタンを増やす

ボタンを 1 つ増やすには、**次の 3 つの作業をもれなく**行います。1 つでも欠けると「ボタンはあるが押しても何も起きない」「処理はあるがボタンが現れない」という状態になります。

1. **画面レイアウト定義に `[BUTTON]` セクションを追加する。** `OnClick` には、押したときに実行する VBA の手続き名（`Btn_` で始まる名前を推奨）を書きます。

    ```
    [BUTTON]
    Cell=N10
    Text=お気に入り
    OnClick=Btn_M08_Favorite
    BackColor=#3366CC
    ForeColor=#FFFFFF
    ```

2. **`OnClick` に書いた名前と同じ `Public Sub` を VBA に実装する。** その画面のブックに対応する入口モジュール（`modEntry...`）に追加します。手続き名は `[BUTTON]` の `OnClick` と完全に一致させます。

3. **Excel の画面シートに実機ボタンを配置し、そのマクロを割り当てる。** 「開発 → 挿入 → フォームコントロール」のボタンを `[BUTTON]` の `Cell` で指定した位置に配置し、右クリック →「マクロの登録」で手続き 2 を選びます。配置の詳しい手順は[導入手順](setup.md)を参照してください。

一覧表の行ごとに増減するボタン（各行の「編集」「削除」など）は、`[BUTTON]` ではなく `[BUTTON_TEMPLATE]` セクションでひな型を宣言します。実際のボタンは、表示するデータが決まったときにツールが自動で生成するため、実機での手配置は不要です。

## 5. 入力項目を増やす

M-05 ナレッジ登録・M-06 ナレッジ修正の入力項目は、画面レイアウト定義ではなく**フォーマット定義**で決まります。項目を増やしたいときは、管理.xlsm の M-03 フォーマット設計画面でフォーマットを編集し、フィールド（入力項目）を追加します。

フォーマットを保存すると、そのフォーマットを使う既存ナレッジへ変更が自動で反映されます。反映の前にバックアップが取られ、確認のメッセージが表示されます。

検索画面（M-08）の検索条件のような、画面に固定された入力欄は、画面レイアウト定義の `[INPUT]` セクションで管理します。

## 6. 変更の反映と確認

画面レイアウト定義ファイルを書き換えたあとは、次の手順で確認します。

1. ファイルを Shift_JIS・改行 CRLF で保存する。
2. 対象のブックを開き直す（または対象画面のシートタブを選び直す）。
3. 変更した画面を実際に開き、表示が意図どおりか目で確認する。
4. ボタンを増やした場合は、ボタンを押して想定どおりの処理が動くことを確認する。

!!! warning "保存時の注意"
    定義ファイルを UTF-8 など別の文字コードで保存すると、日本語が文字化けしたり、画面が読み込めなくなったりします。必ず Shift_JIS で保存してください。

## 7. やってはいけないこと

| 避けること | 理由 |
|---|---|
| 画面レイアウト定義の `[SHEET]` セクションを削除する | `[SHEET]` は必須です。無いと画面を組み立てられません。 |
| `[BUTTON]` を足したのに対応する手続きを実装しない | ボタンを押しても何も起きません。 |
| 定義ファイルを Shift_JIS 以外で保存する | 文字化けや読み込み失敗の原因になります。 |
| `[HEADER]` と `[INPUT]` などで同じセル範囲を重ねて指定する | 範囲が衝突し、画面が崩れます。 |
| バックアップを取らずに編集する | 元に戻せなくなります。編集前に必ずコピーを取ってください。 |
